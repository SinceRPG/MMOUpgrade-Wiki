MMOItems Upgrade System

Build a polished upgrade loop for MMOItems gear.

MMOUpgrade gives players a configurable GUI for upgrading MMOItems with stones, lucky charms, protect charms, level-based success rates, stat scaling, license-based feature limits, and Folia-safe server behavior.

Install

MMOUpgrade is designed for Paper or Folia servers running MMOItems-based gameplay.

Required Plugins

Install MMOItems, MythicLib, and PlaceholderAPI before loading MMOUpgrade.

Recommended Server

Use Java 21 with a Paper or Folia server matching Minecraft 1.21.1.

First Start

Start once to generate configs, then edit them and run /mmoupgrade reload.

Free vs Premium

Free mode keeps the core upgrade system usable. Premium mode removes caps and unlocks per-item control.

Feature Free Premium
Upgrade GUI, stones, charms Available Available
Maximum item level Limited by license-limits.free.max-level Uses full max-level and per-item max level
Success chance cap Limited by license-limits.free.max-success-chance Uses full success-chance.max-percent
Custom MMOItems override stats Ignored except MMOUPGRADE_LEVEL Enabled
Enchantment modifiers Disabled Enabled

Free Limits

license-limits:
  free:
    max-level: 10
    max-success-chance: 75

License Key

license-key: ""

Empty keys run Free mode. Valid keys log Premium License verified during startup or reload.

Commands

Command names, aliases, nodes, and permissions are configurable in commands.yml.

Command Permission Use
/mmoupgrade upgrade mmoupgrade.use Open your own upgrade GUI.
/mmoupgrade upgrade <player> mmoupgrade.admin Open the GUI for another online player.
/mmoupgrade reload mmoupgrade.admin Reload configs and validate the license again.
/mmoupgrade stats mmoupgrade.admin Print registered stat IDs for stat-modifiers.yml.
/mmoupgrade config set-* mmoupgrade.admin Edit simple config values from game or console.

Upgrade Flow

  1. MMOUpgrade checks the player's money using PlaceholderAPI.
  2. The upgrade item and upgrade stone slots are validated.
  3. The current level is read from MMOUPGRADE_LEVEL.
  4. Success chance is calculated from stone, lucky charm, level bonus, and license cap.
  5. On success, the item gains one level and stat modifiers are applied.
  6. On failure, a protect charm may prevent downgrade; otherwise the item may lose one level.
  7. Configured consumables are removed and the economy command runs as console.
  8. Messages and optional level announcements are sent.

GUI

The entire upgrade inventory lives in gui.yml.

title: "<dark_gray>Item Upgrade</dark_gray>"
size: 45

buttons:
  confirm:
    material: LIME_STAINED_GLASS_PANE
    slots: [40]
    cost: 10000
    display: "<green>Upgrade Item</green>"
    lore:
      - "<gray>Cost: <white>{cost}</white></gray>"

Slot Types

confirm, upgrade-stone, lucky-charm, upgrade-item, protect-charm, and filler.

Button Placeholders

Any simple value in the same button section can be used as {key}, such as {cost}, {amount}, or {material}.

Economy

Economy is command-based so it works with many economy plugins.

buttons:
  confirm:
    cost: 10000
    money-placeholder: "%vault_eco_balance_fixed%"
    money-display-placeholder: "%vault_eco_balance_commas%"
    charge-command: "eco take %player_name% {amount}"
Important: money-placeholder must return a plain number. Use formatted values only in money-display-placeholder.

Upgrade Stones and Charms

items.yml only supports MMOItems items. Use MMOITEMS_TYPE:MMOITEMS_ITEM_ID.

upgrade-stones:
  basic:
    id: CONSUMABLE:UPGRADE_STONE_1
    chance: 50

lucky-charms:
  basic:
    id: CONSUMABLE:LUCKY_CHARM_1
    chance: 20

protect-charms:
  basic:
    id: CONSUMABLE:PROTECT_CHARM_1
GroupPurposeNeeds Chance?
upgrade-stonesBase upgrade success chance.Yes
lucky-charmsExtra success chance if inserted.Yes
protect-charmsPrevents downgrade on failure.No

Upgrade Rules

Core upgrade behavior is configured in upgrade.yml.

max-level: 25

license-limits:
  free:
    max-level: 10
    max-success-chance: 75

allowed-types:
  - SWORD
  - ARMOR

failure:
  downgrade: true
  protect-charm-prevents-downgrade: true

Success Formula

stone chance + lucky chance + level bonus, then capped by license mode.

Level Lore

Use level-lore-format with {level} to show the current upgrade level.

MMOItems Custom Stats

MMOUpgrade registers custom stats in MMOItems on startup. Add them to MMOItems item configs exactly like other MMOItems stats, then regenerate or re-give the item.

Premium note: MMOUPGRADE_LEVEL is always used. The per-item override stats below are Premium features.
StatTypeUse
MMOUPGRADE_LEVELDoubleStores the current upgrade level.
MMOUPGRADE_MAX_LEVELDoubleOverrides global max level for one item.
MMOUPGRADE_ENABLEDBooleanAllows or blocks upgrading for one item.
MMOUPGRADE_STONE_CHANCEStringFormula replacing upgrade stone chance.
MMOUPGRADE_LUCKY_CHANCEStringFormula replacing lucky charm bonus chance.
MMOUPGRADE_PROTECT_CHANCEStringFormula for protect charm downgrade prevention chance.

Practical Examples

Premium

Legendary Max Level

MMOUPGRADE_MAX_LEVEL = 50

Use this for rare item lines that should exceed the global cap.

Premium

Enable A Special Bow

MMOUPGRADE_ENABLED = true

Useful when allowed-types does not include BOW.

Premium

Scaling Stone Chance

MMOUPGRADE_STONE_CHANCE =
100 - ({level} * 5)

Level 0 is 100%, level 10 is 50% before modifiers and caps.

Formula Tips

100 - ({level} * 5)
50 + ({level} * 2)
25

Stat Modifiers

Use stat-modifiers.yml to scale MMOItems stats as upgrade levels change.

stat-modifiers:
  attack-damage:
    operation: ADD
    amount: 0.5-1
  mana-cost:
    operation: SUBTRACT
    amount: 0.5-1

Amount Values

Use fixed values like 1 or random ranges like 0.5-1.

Stat IDs

Use lowercase hyphenated IDs. Run /mmoupgrade stats to list registered IDs.

Premium Enchantment Modifiers

enchantments:
  sharpness:
    operation: ADD
    amount: 1-1
    max: 5

The item must already have the enchantment for MMOUpgrade to modify it.

Troubleshooting

Stone or charm is not accepted

Check items.yml uses MMOITEMS_TYPE:MMOITEMS_ITEM_ID, such as CONSUMABLE:UPGRADE_STONE_1.

Item stops at level 10

You are probably in Free mode. Check license-limits.free.max-level or validate a Premium key.

Chance never goes above 75%

Free mode caps final chance with license-limits.free.max-success-chance.

Custom stats do not work

Check Premium mode, exact stat IDs, correct value type, and regenerate or re-give the MMOItems item.

Money is not charged

Check charge-command. It runs as console and must match your economy plugin.

Tooltip shows minecraft keys

That is Minecraft Advanced Tooltips. Toggle it with F3 + H.

License issues: read the console line after Checking LicenseGate license for MMOUpgrade.... It will say whether the key is expired, invalid, unreachable, or valid.
